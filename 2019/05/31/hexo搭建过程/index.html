hexo博客框架搭建过程

2019-05-31
category_1

引言

这是我的第一篇博客, 也是第一次看到这个hexo的框架, 是从YaHei大神那里得知这么个好东西, 因为自己本身就是一个对页面美观有执着追求的人, 因此看到这个简洁的框架和美观的主题后, 毫不犹豫的就想着自己要来搭一个, 毕竟这也是码农一个强有力的工具 ~

现在对hexo的理解并不是特别深入, 大概的感觉就是它不仅仅是一个网站引擎, 而且还可以自己实时的热部署, 与git等的无缝结合, 而且还有很多诸如评论啊搜索啊统计啊什么的功能, 我本来想着git的速度这么慢, 能否把它部署到我的云上, 但后来发现部署到我的云上的话好像也得一堆配置, 上传更新时也得自己配置一些工具和命令, 加之免费云随时就可能没有的风险, 最后还是用了git+微博图床+oss的视频的方式, 自己本地写文档, 然后用hexo -d 的命令部署到git. 我想随着慢慢的使用, 一步步的会更加理解它的配置, 更加理解它的内部机理, 最后能按照自己的想法把博客的内容更好的规划起来.

因此这篇文章就介绍一下hexo的安装配置过程, 万一更换机器后需要重新搭建, 有个技术文档可以参考.

知识准备

  1. md文档是一种标记语言, 比html简单, 并且可以方便的加入图片视频, 可以和html互转, 因此只需将md文件放入到hexo的指定文件夹, 引擎就能将其自动转化成静态html文档.

  2. git提供了资源库供用户使用, 并且提供了丰富的接口可以在远程进行调用, 从而可以将本地网站一键部署, 同时可实现其评论功能.

  3. 前端的node.js是较js更新的前端XX, hexo主要用到了它的环境.(这块还得学习)

安装和使用中的坑们和tips

  1. 当安装完提示致命错误时, 可能是插件没有安装,安装完成消除错误
    npm install hexo-deployer-git –save
  2. 当修改配置文件后, 提示致命错误时, 可能是配置文件的空格被修改了, 冒号后面有且只有一个空格.
  3. 下载主题命令
    git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia
    后面是自定义的文件夹
  4. 文档封面图格式(加载md文档的front中)
    title: Post name
    photos: [
    ["img_url"],
["img_url"]
]
  5. 文档有多个标签时
    tags: [标签1,标签2,标签3]
